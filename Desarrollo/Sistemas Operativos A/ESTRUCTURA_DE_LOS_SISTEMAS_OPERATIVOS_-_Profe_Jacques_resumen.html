🔴 1. SISTEMAS MONOLÍTICOS

Conceptos clave:
  • Todo el sistema operativo funciona como un solo programa gigante.
  • Cada componente del sistema puede llamarse entre sí sin restricciones.
  • Muy difícil de mantener y comprender.

El modelo monolítico es el más clásico y se caracteriza por tener todo el sistema operativo funcionando como un solo programa gigante. Esto significa que cada componente del sistema puede llamarse entre sí sin restricciones, lo que puede ser beneficioso en términos de rendimiento, pero también puede ser un problema en términos de seguridad y mantenimiento.

El problema clave del modelo monolítico es que es muy difícil de mantener y comprender. Si falla un componente del sistema, puede afectar a todo el sistema, lo que puede ser devastador. Por ejemplo, si falla un driver, puede colgar toda la PC.

En resumen, el modelo monolítico es rápido pero peligroso.

🔵 2. SISTEMAS POR CAPAS

Conceptos clave:
  • El sistema se divide en niveles (capas), cada uno con una función.
  • Cada capa solo se comunica con la de abajo o arriba.
  • Más ordenado y estructurado que el modelo monolítico.

El modelo por capas es un enfoque más estructurado y ordenado que el modelo monolítico. En este modelo, el sistema se divide en niveles (capas), cada uno con una función específica. Cada capa solo se comunica con la de abajo o arriba, lo que reduce la complejidad y mejora la seguridad.

El modelo por capas es más fácil de mantener y comprender que el modelo monolítico, pero puede ser más lento y tener un diseño más complejo.

En resumen, el modelo por capas es ordenado y estructurado.

🔴 3. MICROKERNEL

Conceptos clave:
  • Dejar en el kernel solo lo esencial.
  • Los drivers, el sistema de archivos y los servicios se ejecutan como programas de usuario.
  • Más seguro y modular que el modelo por capas.

El modelo microkernel es un enfoque moderno que deja en el kernel solo lo esencial. Esto significa que los drivers, el sistema de archivos y los servicios se ejecutan como programas de usuario, lo que reduce la complejidad y mejora la seguridad.

El modelo microkernel es más seguro y modular que el modelo por capas, pero puede ser más lento debido a la comunicación entre procesos.

En resumen, el modelo microkernel es seguro pero con costo en rendimiento.

🔵 4. MODELO CLIENTE - SERVIDOR

Conceptos clave:
  • Separar en clientes y servidores.
  • Los clientes piden servicios y los servidores los brindan.
  • Todo funciona como “pedido y respuesta”.

El modelo cliente-servidor es un enfoque que separa en clientes y servidores. Los clientes piden servicios y los servidores los brindan, lo que reduce la complejidad y mejora la seguridad.

Todo funciona como “pedido y respuesta”, lo que hace que el modelo cliente-servidor sea fácil de entender y mantener.

En resumen, el modelo cliente-servidor es todo funciona como “pedido y respuesta”.

🔵 5. MÁQUINAS VIRTUALES

Conceptos clave:
  • Simular computadoras dentro de una computadora.
  • Ejecutar varios sistemas operativos.
  • Aislar procesos.

Las máquinas virtuales son un software que simula computadoras dentro de una computadora. Esto permite ejecutar varios sistemas operativos y aislar procesos, lo que reduce la complejidad y mejora la seguridad.

Las máquinas virtuales son muy útiles para probar cosas sin romper el sistema real.

En resumen, las máquinas virtuales son muy útiles para probar cosas sin romper el sistema real.
